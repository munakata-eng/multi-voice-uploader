## なぜ手動操作が重要なのか
Gitの履歴情報（コミット）は、一度保存すると
基本的に修正や削除ができません。

そのため、以下の3つの操作はAIに任せず、
手動で行うことを強く推奨します！

## 危険な操作その1：名前とメールアドレス設定
コミットには必ず「名前」と「メールアドレス」が
セットで公開されます。

必ず手動で設定＆確認を行い、想定通りの内容が
設定されていることをチェックしておきましょう。

```
git config user.name
git config user.email
```

GitHubにはメルアドを隠す設定もあるので、
こちらのnoteも参考にしてみてくださいね。
https://note.com/munakata_souri/n/n5e2e3a94484c#fc301968-69d9-48d3-9f5c-fecfed3b617c

## 危険な操作その2：リポジトリ作成
「リポジトリ」という履歴対象のフォルダを
初期化する手順にも要注意です。

基本的には1つのプログラム = 1つのフォルダ
というように分けて管理するのがオススメですが、
誤って上位のフォルダを指定してしまうと
プライベートな情報が流出する危険があります。

▼ VSCodeでの初期化手順
1. VSCodeを起動
2. プロジェクト用フォルダを開く
3. その状態でリポジトリを初期化

必ずこの手順で初期化することで
誤った場所にリポジトリを作るのを
予防することができます。

## 危険な操作その3：コミット実行
履歴情報を保存する「コミット」の作業も
人間が目視で行うことを強く推奨します。

こちらもVSCode上の機能を使うことで
保存対象のファイルが可視化されるので、

明らかにおかしい量のファイルや
保存してはいけない機密情報を
誤って保存するのを予防できます。

## まとめ

AIプログラミングは非常に便利なのですが
Git操作に関しては以下を徹底しましょう。

1. 名前・アドレス設定 ... 公開情報を慎重に設定
2. リポジトリ作成 ... 対象フォルダを明確に指定  
3. コミット実行 ... 実行前に対象ファイルを確認

面倒に感じるかもしれませんが、
安全にプログラミングを扱うために
手動で作業するよう心がけていきましょう！
