## OGPって何？

XやDiscord、NoteなどでURLを投稿すると、自動的にそのサイトの説明文や画像が視覚的に表示されることがあります。
これがOGP（Open Graph Protocol）という仕組みです。

OGPはサイト作成者側が設定しないと表示されず、これがないと
・サイトの説明が表示されず中身がすぐにわからない
・怪しいリンクだと思われてクリックに繋がらない
・見た目がイマイチで魅力を伝えられない
ということが起き得るので、サイト作成時にはぜひ設定してみましょう！

## OGPの基本設定項目

OGPで設定する主な内容は以下の4つです。

1. **サイトのタイトル**
2. **説明文**
3. **表示する画像のURL**
4. **サイト自体のURL**

ここにあるように設定には公開後のURLが必要になるので、一度サイトを公開してURLを確定させてから
その後にOGPを設定するような手順がおすすめです。


## キャッシュに注意！
OGPの表示のときには負荷軽減のため「キャッシュ」という仕組みで前回の情報を保存することが多いです。

そのため、OGPの内容を変更してもすぐに最新の状態が反映されないことが多いので覚えておきしましょう。

ちょっとしたテクニックですが、URLの後半にパラメータを設定することで
最新の情報が設定されているかを確認することもできます。
```
https://example.com/?check=1
```

このように「?」以降を付けると、別のURLとして認識され新しい情報を取得してくれます。


あくまでチェック用なので、元のURLでの反映のためにはしばらく時間をあけるか、
キャッシュ削除の仕組みが用意されているものであればそちらを利用してみてください。


## AIに依頼するときのコツ

AIにOGP設定を依頼する場合は、具体的に情報を伝えることが重要です：

```
以下の内容でOGPを設定してください：
- タイトル：「○○について」
- 説明文：「○○の基本的な使い方を解説します」
- 画像：「/images/ogp.png」のパスにある画像を使用
- URL：「https://example.com」
```

特に画像に関しては場所を的確に指示しないとうまく表示されない可能性があるので
うまくいかないときには根気よく試してみましょう！

## まとめ

OGPは、SNSでURLをシェアしたときに表示される「プレビュー」を設定する仕組みです。

設定することでサイトの魅力や内容をわかりやすく伝えることができるので、
自分でサイトを作ったときには積極的に設定してみてくださいね。
