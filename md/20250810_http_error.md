## HTTPエラーコードの基本構造

Webページを表示したときのエラー番号は
「HTTPエラーコード」と呼ばれ、
100番台〜500番台の3桁の数字で構成されます

- **100番台**: 情報レスポンス
- **200番台**: 成功（正常処理）
- **300番台**: リダイレクト
- **400番台**: クライアントエラー
- **500番台**: サーバーエラー

今回は問題発生時によく目にする
**400番台**と**500番台**について
詳しく解説していきますね。

## 400番台：ユーザ側に問題あり

何かしらアクセス方法の間違いの可能性があるので
URLや権限を再確認する必要があります。

### 404 Not Found

「指定されたURLのページが存在しません」
という意味のエラーで、よく見かけるエラーです。

このエラーが出たらまずは「URLが正しいかな？」
ということを確認してみましょう。

### 403 Forbidden

「ページはあるけどあなたに閲覧権限がありません」
という意味になります。

適切な権限を持っているかを今一度確認して、
場合によっては管理者に問い合わせてみましょう。

## 500番台：サーバー側に問題あり

ユーザとしてアクセスしている場合は
少し時間を置いて解決を待つしか無いですが、
自分が作っているサイトの場合は
早急な原因究明と修正が必要となります。

### 500 Internal Server Error
「サーバー側で何らかのエラーが発生しています」

アプリケーション内のエラーで、プログラムのバグや
データベースの接続数の急増によって発生します。

### 502 Bad Gateway
「サーバー間の通信でエラーが発生しました」

通信に関わるエラーで、アクセスが集中した時や、
インフラに障害が発生したときに発生します。

### 503 Service Unavailable
「サーバーが一時的に利用できません」

メンテナンス中や一時的な高負荷状態で
発生することが多いです。

## まとめ

エラー番号の意味を知っておくと、問題の切り分けや
開発時のトラブル対応に大きく役立ちます。

自分が作っているサービスでエラーが出たときにも
慌てずに対処できるようにしましょうね。
