## Gitとは？プログラムの「セーブデータ」

Gitとはファイルの変更履歴を保存できる「バージョン管理システム」の一つです。

…といってもよくわからないと思うのですが、ゲームのセーブデータのような仕組みで、
・プログラムの状態を名前をつけて保存
・過去の状態にいつでも戻せる
という機能が備わっています。


AIにお願いしていると起きがちな
・「プログラムが壊れて動かなくなった！」
・「デザインがおかしくなった！」
といった状況に陥っても、過去のバージョンに戻すことができるようになります！



## GitHubとは？セーブデータの「保管庫」
名前が似ているのですが、Gitはセーブデータを保存できる仕組みのことを指し、GitHubはそのセーブデータをアップロードできる「保管庫」のようなWebサービスです。

自分のパソコン内にセーブデータを残しておくだけだと、パソコンが壊れるとデータが消えちゃうのですが、GitHubにアップロードしておくことで、いつでもデータが復元できるようになります！

また、この保管庫にアップロードすることで、作ったWebページのプログラムを一般公開できる仕組みも備わっています。

バックアップを行いつつWebサイト公開もできるということで、AIプログラミングを行う方にとっては非常に便利なサービスとなります。



## まとめ
**Git** = 手元でファイル履歴を管理する仕組み
**GitHub** = その履歴をWeb上に保存する保管庫サービス
