## インターネット上の「住所」の扱い

ポート番号について紹介する前に、
インターネット上の住所の扱いについて
簡単におさらいしていきますね。

### IPアドレス

3桁の数字が4つ組み合わさった番号で、
これによってインターネット上の住所を
特定することができます。

例: `172.217.175.14`

### ドメインとIPアドレス

IPアドレスを人間が覚えるのは難しいので、
- `google.com`
- `yahoo.co.jp`
のような覚えやすい文字列（ドメイン）で
Webサイトにアクセスするのが一般的です。

アクセス時にはドメインに紐づいたIPアドレスを
検索する仕組みが裏側で動いているため、
意識せずに見たいサイトが閲覧できるのです。

## ポート番号とは

IPアドレスをネット上の「家の住所」だとすると
ポート番号は「部屋番号」のようなものです。

URLの後ろに「:」+「数字」を付けて表現され
「http://localhost:3000」
のように番号を指定してアクセスします。

ポート番号があるおかげで、同じIPアドレスに対し
複数サービスを同時公開することができるのです。

## 開発者は避けて通れないポート番号

ユーザとしてサイト回遊しているときには
特に意識しなくても大丈夫なのですが、
開発者としてサイトを作成しているときには
ポート番号は避けて通れません。


### 開発時の部屋分け

自分のパソコンも1つの「家」の扱いなので
その中で複数のサービスを立ち上げるときには
ポート番号で部屋分けする必要があります。

例:
localhost:5500 ... HTMLページ
localhost:3000 ... React
localhost:8080 ... APIサーバ


### ポート番号の競合に注意

ただし、ここで気をつけなければならないのが
「同じポート番号は複数のサービスで使えない」
という点です。

特にAIプログラミングを行っている場面では
このようなエラーを見たことがある方も
いらっしゃるかもしれません。

```
Error: Port 3000 is already in use
```

このようなエラーが出たときには、
1. 起動中のサービスを停止
2. 別のポート番号を使用
のいずれかの対応が必要となります。

原因さえわかっていればAIに指示して対策も
簡単ですので、ぜひ覚えておきましょう！
