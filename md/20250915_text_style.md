## テキスト形式って？

コンピュータがデータをやり取りするためには
ある一定の規則性が必要です。

ただし、作成するのは人になるので、
「コンピュータも読みやすく人間も理解しやすい」
中間的な形式が必要となります。

これが「テキスト形式」です。
今回はよく使われるテキスト形式を
5つ紹介しますね。

## 1. XML（エックスエムエル）

<person>
  <name>太郎</name>
  <age>30</age>
</person>

タグで情報を囲む形式で、
HTMLの大元となる考え方です。

構造がわかりやすく
親子関係を表現できるのですが、
同じタグを複数書く必要があり
全体が長くなりがちです。

## 2. CSV（シーエスブイ）

名前,年齢,職業
太郎,30,エンジニア
花子,25,デザイナー

カンマで区切るシンプルな形式で
表形式のデータに特化しています。

非常にシンプルに読み書きできるのですが、
親子関係を表現することはできません。

## 3. YAML（ヤムル）

person:
  name: 太郎
  age: 30
  skills:
    - Python
    - JavaScript


インデント(スペース数)で階層を表現する方法で、
少ない文字数で複雑な構造を表すことができます。

書き方ルールが曖昧なため、スペース数のずれ等
手動でミスが起きやすいのが弱点ではあります。

## 4. JSON（ジェイソン）

{
  "name": "太郎",
  "age": 30,
  "skills": ["Python", "JavaScript"]
}

中括弧の中でデータを表現する形式で
人間もコンピュータも理解しやすい
というメリットがあります。

データ量も少なく済むため、
APIのやり取りにも使われています。

## 5. Markdown（マークダウン）

# 見出し1
## 見出し2
**太字**
- 箇条書き1
- 箇条書き2

記号を使って文書装飾する形式で、
ブログ記事の執筆や、AIのルール設定に
広く使われます。

厳密なデータやり取りには不向きですが、
曖昧さがあっても大丈夫なAIの分野で
採用されることが多いです。

## まとめ

今回は、コンピュータとやりとりするための
5つのテキスト形式を紹介しました。

1. XML - タグベースの構造表現
2. CSV - シンプルな表形式データ
3. YAML - インデントベースの設定
4. JSON - 現在の主流形式
5. Markdown - 文書装飾に特化
