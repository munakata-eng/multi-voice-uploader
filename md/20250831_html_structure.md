## Webサイトを構成する3つの要素

まずは基本のおさらいから。
Webサイトは以下の3つで構成されています。

### HTML：全体の枠組み
- どこに何を配置するかを決める
- サイトの**骨組み**となる部分

### CSS：デザイン
- 色、文字サイズ、余白を設定
- サイトの**見た目**を整える

### JavaScript：動き
- 要素の移動、フェード効果
- データの取得など**機能**を追加


今回の記事ではこの中でも最も基本となる
**HTML**について解説していきます。

## HTMLの基本構造

HTMLは < と > で囲われた
**タグ**という要素で構成されています。

一番外側は<html>というタグで囲われていて、
それぞれ役割の異なるタグを組み合わせて
1つのサイトが作られています。


### 「要素」: 特定の役割をもつ部品

HTMLでは < > のタグ内に書く英語によって、
異なる役割をもたせることができ、
この役割をもった部品を「要素」と呼びます。

□ 要素の例
- <h1> ... 見出し
- <a> ... リンク
- <button> ... ボタン
- <p> ... 段落
- <li> ... 箇条書き

### 要素に名前をつける2つの方法

プログラムから各要素に指示を与えるために
で名前をつける仕組みが2つあります。

① id
・そのページ内で「唯一無二」の名前
・必ず1つの要素を取得することができる

② class
・要素の「グループ分け」に使用する名前
・同じページ内で複数つけることができる

## AIプログラミングでの活用方法

では具体的にどんな指示が行えるかを見ていきます。

例）ヘッダー内のボタン
<div id="header" class="main-nav">
<button class="menu">メニュー</button>
</div>

□ 悪い例
「メニューボタンを押して」

□ 良い例
「idが"header"のdiv要素の中にある
classが"menu"のメニューボタンを押して」


もちろん前者でもAIが前後関係を理解して
うまくいくことはありますが、
「メニュー」ボタンが複数ある場合には
失敗する可能性が上がってしまいます。
