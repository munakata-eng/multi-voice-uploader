# プログラムが「うまく動かない」時にまずやるべきこと
こんにちは、むなかたです。

今回は、プログラムが動かない時に
どんな順番で解決するべきかを
簡単に紹介していきます。

同じ内容は音声でも配信しているので、
お好きな方法でインプットしてくださいね。

## 1. まずはエラーを見る

一番の手がかりは何と言ってもエラー文です。

エラー画面は怖いイメージがありますが
開発のときには有力な手がかりになるので
有効に活用しましょう。

- 画面上のエラーメッセージ
- エディター／実行コンソール
- ブラウザの開発者ツール（Console）

のいずれかでエラーが出ていた場合は
内容をコピーしてエラー内容を
AIににそのまま投げてあげればOKです。

## 2. 「いつから」問題が出たかを考える

新機能追加や大きな変更後など
動かなくなったタイミングを
知ることも重要です。

簡単なのはその変更を一度もとに戻すこと。
Gitという仕組みを使って保存していれば
確実に前回の状態に戻せるので
ぜひ活用していきましょう。

## 3. エラーが発生する条件を切り分け

常に起きるエラーではなく、
特定の条件下でだけ
発生するものもあります。

その場合は「5W1H」の考え方で、
なるべく細かく再現条件を絞り込みましょう。

- PCはOKだけどスマホはNG
- WindowsはOKだけどMacだとNG
- この手順を踏むとNG　など

何をどうしたときにどんなエラーが出るか、
これを意識することが重要です。

## 4. 最終手段は作り直し
どうしても直らない場合は、
潔く新規に作り直すのも有効です。

人の手で作った場合には大変ですが
AIプログラムでは作り直しは簡単。

これまで頑張ったのに…
という気持ちは一度切り替えて
次のものを作ったほうが
案外解決が早いケースが多いです。

## まとめ

作ったプログラムが上手く動かないときには
順を追って原因を探っていきましょう。

どうしてもわからないときには
潔く作り直すのも有効です。

バグを必要以上に恐れすぎず
楽しく開発を進めていきましょう！
